# LP コンテンツ構成案

> ステータス: **ドラフト**。LP に載せるセクションのたたき台。実装時に取捨選択する。

紹介ページとして 1 ページ (シングルページ) で完結させる想定。上から下へ読むだけで
「何のゲームか → 面白さ → プレイ」までたどり着ける構成にする。

## セクション案

1. **ヒーロー (ファーストビュー)**
   - キャッチコピー + 一言説明
   - 「今すぐプレイ」ボタン (web 版へ) / 「GitHub を見る」ボタン
   - ゲーム画面のスクリーンショットまたは短い GIF

2. **コンセプト**
   - 「直接操作しない。AI ルールを組んで戦わせる」という核を説明
   - 設計 → 観察 → デバッグ → 改善のサイクルが面白さ、という訴求

3. **遊び方 (3 ステップ)**
   - ① ルールを組む (ガンビット式) → ② 自動戦闘を見る → ③ ログで分析して直す

4. **特徴 / 魅力**
   - ガンビット式 AI 設定 / 決定論シミュレーション (同じ条件なら同じ結果) /
     戦闘ログによる分析

5. **スクリーンショット or デモ**
   - ルールエディタ・戦闘画面・ログビューのキャプチャ

6. **ロードマップ (任意)**
   - 現在は MVP、今後 2on2・障害物・複数ユニット等 (本体 SPEC のロードマップに連動)

7. **フッター / CTA**
   - プレイ導線・GitHub・ライセンス・クレジット

## 用意したいアセット

- ゲーム画面のスクリーンショット数点 (本体 web 版から取得)
- できれば 5〜10 秒のプレイ GIF / 動画
- OGP 画像 (SNS シェア用)

## コピーの素材

- ゲーム概要・コンセプト・ロードマップは本体の
  [SPEC.md](https://github.com/keroway/code-tactics/blob/main/SPEC.md) が正典。
  ここから言い回しを引いて LP 向けに短くする。
