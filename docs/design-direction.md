# デザイン検討のたたき台

> ステータス: **ドラフト**。デザインを確定するための叩き台。決め打ちせず選択肢を
> 並べる。実装フェーズで 1 案に収束させる。コンテンツ構成は
> [content-plan.md](./content-plan.md) と対で読む。

## ブランドトーン

ゲーム性そのものを世界観の手がかりにする:

- **ガンビット式 AI 設定** → ルールを「組む」エンジニアリング感
- **決定論シミュレーション** (同じ条件なら同じ結果) → 再現性・論理性・信頼感
- **戦闘ログによる分析・改善** → 観察と解析、計器的・コンソール的な雰囲気

→ キーワードは **戦術的 / エンジニアリング的 / 解析的**。
かわいさやファンタジーよりも、「設計してデバッグする楽しさ」を前面に出す方向。

## 配色案 (候補)

決め打ちせず 2〜3 方向を提示。実装時に 1 つへ収束させる。

1. **ダークターミナル系** — 暗い背景 + 等幅フォントのログ表現 + 単色アクセント
   (緑/シアン)。コンソール・コードエディタを想起させ、ゲーム性と直結。
2. **テック・ブループリント系** — 濃紺ベース + グリッド/設計図的な装飾 +
   明るいアクセント。「戦術を設計する」イメージ。
3. **ライト + アクセント系** — 明るい背景で可読性最優先、要所に鮮やかな 1 色。
   万人向けで親しみやすいが、ゲームの硬派な雰囲気は出にくい。

いずれもアクセントは 1〜2 色に絞り、CTA (「今すぐプレイ」) を最も目立たせる。

## タイポグラフィ

- **見出し**: 力強いサンセリフ。短いキャッチコピーを大きく。
- **本文**: 可読性重視のサンセリフ (日本語フォントの選定も要検討)。
- **等幅**: ログ・ルール・コード風の演出に使用。ゲームの「プログラマブル」感を
  視覚的に補強する重要要素。

## レイアウト

[content-plan.md](./content-plan.md) のセクション (ヒーロー → コンセプト →
遊び方 3 ステップ → 特徴 → スクショ/デモ → ロードマップ → フッター CTA) を、
上から下へ読むだけで「何のゲームか → 面白さ → プレイ」へ到達する 1 カラム構成に。

- ファーストビューでキャッチコピー + 主要 CTA + ゲーム画面を即提示。
- 「遊び方 3 ステップ」は番号付きカードで横並び (モバイルは縦積み)。
- CTA はヒーローとフッターの最低 2 箇所に置く。

## モーション

- 控えめなスクロール連動フェードイン程度に留め、過剰な演出は避ける
  (静的 LP の速さを損なわない)。
- デモは 5〜10 秒の GIF / 動画。自動再生する場合はミュート + ループ + 軽量化。

## OGP / SNS シェア

- OGP 画像 (シェア時のサムネ) を 1 枚用意。ゲーム画面 + ロゴ + キャッチを 1 枚に。
- title / description / og:image を適切に設定 (Astro の `<head>` で管理)。

## アクセシビリティ

- 本文・CTA は十分なコントラスト比を確保 (ダーク系採用時は特に注意)。
- CTA・ナビはキーボード操作可能でフォーカスリングを残す。
- 画像に alt、動画/GIF には説明を添える。
- アニメーションは `prefers-reduced-motion` を尊重する。

## 必要アセット

[content-plan.md](./content-plan.md#用意したいアセット) と共通:
ゲーム画面のスクリーンショット数点 / 5〜10 秒のプレイ GIF・動画 / OGP 画像。
本体 web 版から取得する。
